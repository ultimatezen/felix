VXPLib Code Samples

Code Samples for Microsoft Visual C++ 6 and .NET

All code examples presented here were originally designed to run under Microsoft Visual C++ 6.
However, they are all portable automatically into Microsoft Visual C++ .NET

SamplePurpose
Custom Tooltips Shows how to attach tooltips to objects that do not have an ID or Window Handle. For instance, how to attach tooltps to items in a List Control or Tree Control.
Static Controls Example of embedding controls in QHTML when WIDTH and HEIGHT of all controls can be set within QHTML
Dynamic Controls Example of embedding controls in QHTML when WIDTH and HEIGHT of some controls must be calculated dynamically.
Static Text A simple class CVXPStatic derived from CStatic to draw its contents using QHTML. This class supports all the standard properties of the STATIC control, such as alignments and disable state. Additionally, it allows to draw sunken QHTML using method DrawState of the IVXPQuickHTML interface.
Simple Tooltip
Would you spend just 10 seconds to think of how large on your opinion could be a pure Windows API application written in C++ that would be creating a window, attaching a fancy tooltip to it, and displaying this all?
One hundred or more?
If you use VXPLib the answer is 10 lines!
Run the example, then freeze the mouse cursor over the window for just a moment to cause the toolip to appear.
HyperLinks
This example is for VC++ developers only. It features class CHyperLink when it can be used on its own to support standard hyperlinks, and when it is overriden to draw hyperlinks using Quick HTML.
XP Menus
See how easily you can use IVXPQuickHTML to draw menu items the same way as in Windows XP.
Additionally, the example shows how to attach tooltips to menu items, and how to draw in the main window using Quick HTML parser.
Parser Test
This program is an ultimate tool to work and experiment with the IVXPQuickHTML parser. It allows the following:
  • Enter any QHTML text in any format; The project can be compiled in both ASCII and Unicode modes.
  • Set all parameters of the parser such as default resource file, list of search directories and default code page for tag <TXT>
  • See the output result with such information as output size, recalculation and displaying time, and list of all events fired by the parser while processing the document.
  • Saving and loading QHTML documents using format UTF8 in the Unicode mode, and using format ASCII in the ASCII mode.
Note: Avoid running this example from VC++ environment, because it happens to have some mystique problems with Common Dialogs on W2K: FileOpen and FileSave, they can work just way too slowly. Whoever tells me what's wrong there - thank you in advance.

All examples for C++ were implemented using a header file VXPLib.h for easier access to methods and properties of the COM library. Each interface implemented in the VXPLib component has a C++ class in VXPLib.h with the same name.

10 Interfaces:

  • CVXPQuickHTML - represents interface IVXPQuickHTML and its events.
  • CVXPTooltip - represents interface IVXPTooltip and its events.
  • CVXPTooltipManager - represents interface IVXPTooltipManager and its events.
  • CVXPFont - represents interface IVXPFont
  • CVXPMargins - represents interface IVXPMargins
  • CVXPSound - represents interface IVXPSound
  • CVXPSettings - represents interface IVXPSettings
  • CVXPCmdToolList - represents interface IVXPCmdToolList
  • CVXPWndToolList - represents interface IVXPWndToolList
  • CVXPMenuHook - represents interface IVXPMenuHook and its events.

The main advantage of using this header file is that it allows to handle all COM events supported by VXPLib using Active Template Library.
